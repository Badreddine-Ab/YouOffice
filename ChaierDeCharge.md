Cahier de Charge : Gestion des Cabinets
Introduction : Le projet consiste en développement d'une application de gestion des cabinets médicaux pour les patients, les réceptionnistes, les docteurs et le super admin. L'application permettra à chaque rôle d'accomplir les tâches nécessaires pour une gestion efficace du cabinet.
Technologies utilisées:
•	Urql
•	Express
•	ReactJS
•	PostgreSQL
•	Native React
Fonctionnalités pour les patients :
•	Accès au dossier médical
•	Historique des paiements et rendez-vous futurs
•	Paiement en ligne pour les consultations
Fonctionnalités pour les réceptionnistes :
•	Planification des rendez-vous pour les patients
•	Marquage de la présence le matin
•	Suivi des paiements
•	Impression de reçus de paiement avec logo et informations légales du cabinet
Fonctionnalités pour les docteurs :
•	Accès aux dossiers médicaux des patients
•	Planification des rendez-vous
•	Accès à l'API MedlinePlus Connect pour suggérer des médicaments
•	Impression d'ordonnances avec les médicaments recommandés et les informations du cabinet
Fonctionnalités pour le super admin :
•	Gestion complète de l'application
•	Ajout, mise à jour et suppression de données/utilisateurs
Priorités :
•	Version web en premier
•	Version mobile par la suite
Plan de développement :
1.	Étude de faisabilité
2.	Conception de l'interface utilisateur
3.	Développement de la base de données
4.	Développement de la partie back-end
5.	Développement de la partie front-end
6.	Tests unitaires et intégraux
7.	Mise en production
Conclusion : Le projet de développement d'une application de gestion des cabinets médicaux permettra une gestion plus efficace et simplifiée pour les patients, les réceptionnistes, les docteurs et le super admin. Les fonctionnalités proposées et les technologies utilisées assureront une expérience utilisateur optimale.
